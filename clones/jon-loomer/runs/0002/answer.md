# Answer (reasoning as Jon Loomer) — run 0002 — How Jon tests creatives

Core: use the in-ad-set **creative testing tool** as the spine of the process [e_s074_01], start new ads with a test [e_s046_01], test small in **staged batches**, and read for **themes**, not single winners [e_s055_05]. Only test when results could be better [e_s079_01].

**Why the tool over the old way.** Historically you ran a separate test campaign/ad set then moved winners — but there's no guarantee a winner still performs after it's moved, plus extra campaigns and moving parts [e_s074_03]. The tool keeps the ads in the ad set they'll live in, and gives you real data instead of guessing by stopping/starting ads [e_s074_04].

**How it works.** A button starts a test of up to 5 (now 10) new ads — you can't test existing ads [e_s074_02][e_s097_01]. You set the budget share, duration, and success metric; budget is spent evenly with no overlap, like a normal A/B test [e_s074_02].

**The process.** Whenever Jon publishes new ads, he starts them with a test created *inside the active ad set* — not a separate testing campaign [e_s046_01]. A new ad set starts with a small handful (2–5 ads), not 20–30 at once [e_s074_05]. For a brand-new campaign, manually set the test to 100% of budget instead of Meta's default ~20%, since the only ads there are your test ads [e_s046_02]. Reasons to start new ads with a test: you see how each ad does on equal budget (no wondering weeks later about an ad that got no impressions) [e_s046_03], and you avoid the messy duplicate-and-recreate you're forced into if you try to test an active ad later [e_s046_04].

**Staged / "stacking" approach.** Phase one: one ad set, a general theme, five variations (e.g. five color combos sharing the same copy), a 7-day test so each version actually gets budget, not just the algorithm's pick [e_s068_04]. After the test ends, let ads run without a test for at least a week — if aggregate results are great, leave them alone; if bad or just okay, start another test in the *same* ad set with completely different ads (new visuals, messaging, personas) [e_s074_07]. Phase two ("stacking"): learn from set one to build a uniquely different second set, test it within the existing ad set, then repeat [e_s068_06][e_s090_04]. This is creative diversity done in phases, slowly and strategically, via the testing tool [e_s068_03].

**Make tests meaningful.** Testing is inefficient by nature (forced even budget, no overlap), so make sure it runs long enough and spends enough — five ads at $20/day to sell something won't generate enough volume to learn [e_s074_06]. Don't read the test as "keep the single best primary text"; read it for the themes that worked, and use those as the starting point for the next batch — not a reason to switch off your other ads [e_s055_05].

**Don't over-test.** Most advertisers shouldn't take Meta's "dump tons of diverse creative" literally — with a small budget, throwing 10–20+ ads in at once is wasted effort because Meta concentrates budget on a few anyway [e_s068_02]. You may not need more than a handful if early results are good [e_s068_07]. And a test always pulls budget away from what's working, so if you're getting amazing results in aggregate, there's no great reason to test — take the win and let the winners run [e_s079_01][e_s079_02][e_s079_03].

*Confidence: High. This is a documented, repeatedly-stated 2025–2026 method built around Meta's creative testing tool. Caveat: the tool itself is new and evolving (variation cap moved 5→10; budget-distribution bugs reported), so specifics may shift.*
